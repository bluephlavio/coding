# Come usare questo repository

Questo file ti guiderà passo dopo passo su come lavorare con il repository durante il corso.

## 1. Navigare nel Repository

Il repository è organizzato in **cartelle di lezione**. Ogni cartella (`lessonX_nome_lezione/`) contiene:
- Un file `README.md` con le istruzioni specifiche per quella lezione.
- File vuoti o semi-vuoti dove scrivere il tuo codice per gli esercizi.

### Come trovare il materiale:
1. Vai alla cartella della lezione che stai seguendo.
2. Leggi attentamente il `README.md` per comprendere gli obiettivi e le istruzioni.
3. Completa gli esercizi nei file vuoti forniti.

---

## 2. Come lavorare sugli esercizi

### Iniziare a lavorare:
1. Apri il repository con **GitHub Codespaces** (vedi [README.md](README.md) per i dettagli).
2. Naviga nella cartella della lezione e apri i file degli esercizi.
3. Scrivi il tuo codice nei file vuoti o semi-vuoti forniti.

### Salvare il tuo lavoro:
Per salvare i tuoi progressi, nel terminale di Codespaces, digita:
   ```bash
   git add .
   git commit -m "Messaggio descrittivo del tuo progresso"
   git push
   ```
Questo aggiornerà il tuo repository personale su GitHub con i tuoi progressi.

## 3. Ottenere feedback sul tuo lavoro

Quando hai completato uno o più esercizi, puoi chiedere feedback all'insegnante aprendo una **Pull Request (PR)**.

### Come aprire una Pull Request:
1. Vai al tuo repository forkato su GitHub.
2. Clicca su **Pull requests**.
3. Clicca su **New pull request**.
4. Seleziona il branch `main` del tuo repository personale come **base** e il branch `main` del repository originale del corso come **compare**.
5. Inserisci un titolo descrittivo e un messaggio per la PR, quindi clicca su **Create pull request**.

L'insegnante potrà vedere il tuo lavoro e fornire feedback direttamente sulla PR.

---

## 4. Aggiornare il tuo fork con il repository originale

Durante il corso, il repository principale potrebbe essere aggiornato con nuovo materiale o correzioni. Per sincronizzare il tuo fork con il repository principale:

1. Vai al tuo repository personale su GitHub.
2. Clicca su **Sync fork** (pulsante disponibile nella pagina principale del tuo repository).
3. Conferma cliccando su **Update branch**.

Questo assicura che tu abbia sempre l'ultima versione del materiale.
